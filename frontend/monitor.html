🚨 Monitoring

Audit & Threat Monitoring

Every security event across your apps is logged automatically. Query the tamper-evident audit trail, view active threat alerts, and mark incidents as resolved after investigation.

What gets logged automatically: key issued · key rotated · key revoked · auth success · auth failure · replay detected · brute force detected · revoked key use.
Every event includes a timestamp, app ID, and the client IP address where available.
Audit Log
Query the tamper-evident log of all security events for your account. Filter by app ID to see events for a specific platform. Events are returned newest-first.
POST /quantum/audit/query
—
Threat Alerts
Get active threat alerts for your apps. Three threat types are detected automatically:

● brute_force — 5+ failed verifications in 60s
● replay — same key value used more than once
● revoked_key_use — attempt to use a revoked key
POST /quantum/monitor/alerts
—
Resolve Threat
Mark a threat event as resolved after you have investigated and addressed it. Resolved threats are excluded from the default alerts view but remain in the audit log permanently.
POST /quantum/monitor/resolve/{threat_id}
—
Severity levels:
🔴 critical — brute force attack detected (immediate action required)
🟠 high — replay attack or revoked key misuse
🟡 medium — unusual access patterns
🟢 low — informational anomalies
