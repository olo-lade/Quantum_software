🗝️ Key Management

Managed Key Lifecycle

Issue quantum-random keys to your apps, rotate them on a schedule, revoke compromised keys instantly, and verify them with automatic threat detection — all with a full audit trail.

How it works: Issue a key for your app ID → use key_id + key_value to authenticate requests → verify calls automatically detect replay attacks, brute force, and revoked key misuse → rotate or revoke at any time.
Issue Key
Generate and store a quantum-random key for a client app. Optionally set a TTL — the key automatically expires after the given number of hours.
POST /quantum/keys/issue
—

Copy the key_id — you'll need it for rotate, revoke, and verify.

Rotate Key
Replace a live key with a fresh quantum-random one. The previous key value is preserved in the response for a graceful transition period — your app can switch over without downtime.
POST /quantum/keys/{key_id}/rotate
—
Revoke Key
Permanently revoke a key. Any future verification attempt using a revoked key is automatically flagged as a high severity threat event and written to the audit log.
DELETE /quantum/keys/{key_id}/revoke
—
Verify Key
Verify a managed key and automatically detect threats. Returns valid: true/false plus a reason. Triggers alerts for: replay attacks, brute force (5+ failures in 60s), revoked key use, and expired keys.
POST /quantum/keys/verify
—
Threat detection summary:
🔴 brute_force — 5+ failed verifications within 60 seconds from the same source
🟠 replay — the same key value is submitted more than once
🟠 revoked_key_use — any attempt to use a key that has been revoked

All threats are written to the audit log and appear in Monitoring → Alerts.
