🔐 Cryptography

Quantum Cryptography

True quantum randomness for key generation, unbreakable One-Time Pad encryption, randomness quality auditing, and BB84 quantum key distribution — all via simple REST calls.

Why quantum randomness? Classical computers generate pseudo-random numbers using algorithms — predictable if you know the seed. Quantum circuits use physical superposition, making the output fundamentally unpredictable and impossible to reproduce.
Key Generation
Generate a cryptographic key using Hadamard superposition. Each qubit is placed in a perfect 50/50 state before measurement — true quantum randomness.
POST /quantum/crypto/keygen
—
Entropy Scoring
Audit the randomness quality of the quantum backend. Runs the circuit multiple times and calculates Shannon entropy on the outcome distribution. Score of 1.0 = perfect randomness.
POST /quantum/crypto/entropy
—
OTP Encrypt
Encrypt a message using a quantum-random One-Time Pad. A quantum key the exact length of your message is generated and XOR'd byte-by-byte. Mathematically unbreakable when the key is truly random.
POST /quantum/crypto/otp/encrypt
—

⚠️ Copy the key from the response — it is never stored server-side.

OTP Decrypt
Decrypt a quantum OTP ciphertext using the original key returned during encryption. XOR reversal is applied byte-by-byte to recover the original plaintext.
POST /quantum/crypto/otp/decrypt
—
BB84 Key Distribution
Simulate the BB84 Quantum Key Distribution protocol. Alice and Bob exchange qubits using random bases. Matching bases produce a shared secret key. Any eavesdropper disturbs the quantum state — detectable via error rate above 10%.
POST /quantum/crypto/bb84
